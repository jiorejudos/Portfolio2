Marco Mobile App

Marco is a road trip planning app. It was born out of many trips that I have taken and struggled to find a convenient restaurant along my way. Many map apps search around a certain area, so you don't necessarily find something in the direction the driver is traveling. Marco searches along the route. The user can also save the itinerary for later. Users also get detailed information of the businesses along with reviews.

User surveys help to understand what users want and what type of elements to incorporate into the app.

User Personas

User Personas are a good way to get a better understanding of the users' goals, behaviors and how they will interact with the application.

Competitive Analysis

A competitive analysis is a good way to find out about the competitors and what needs improved upon. I looked at Google, Yelp and Foursquare.

Strengths

In all three apps, it is easy for a user to search for what they need. Google is great at giving directions. Yelp has detailed reviews. Foursquare has lots of filters in the search to help users find what they want.

Weaknesses

Only Google allows users to search along a planned route, but they can’t easily save itineraries.

Opportunities

An app that allows a user to save a planned road trip itinerary would be useful.

Threats

An app that specializes in searching for locations on a road trip, where you can easily save the itinerary has a competitive edge.

User Flows

For the Marco app, the user needs to be able to search for businesses, sign up for an account, find business reviews, write reviews and save their itineraries. I created User Flows for these actions, which helped me to design a good user experience.

Style Choices

This app is all about exploring as you go on a roadtrip, so naming it after the explorer, Marco Polo is fitting. Marco is a strong name, and it performed well on user surveys. The color choices go along with the colors on a sunset drive. Open Sans is a good font for its excellent legibility on mobile interfaces.

User Testing

Throughout the process, I did user testing. My first test was to have users search along a route for a restaurant. The users said they were not sure how to get to the search along route feature, so the text “search along route” was added to the search field. The user experience could still be improved, so in the final design an overlay card is at the bottom where you can easily add a destination.

Final Design

Users were then asked to try to review a restaurant. The users said it was hard to find “Reviews” at the bottom of the page. Based on user feedback, the “reviews” link is in the middle of the screen. This makes it clearer for the testers. In the final version, there is one more slight change to the design. The “Reviews” link is now “My Reviews”, which makes it even clearer for the users.

Final Design

I also did alot of testing for the button placement on the Find Location page. In the first version, everything is at the top of the screen. So in the final version, everything is in the middle. This tested well on users.

Prototype

The Marco user needs to get directions and search along a route. The user also needs to view information about a business and see business reviews. The app also includes an optional sign in process, so users can save their itinerary.

Project Lessons

While designing this app, I learned to spend more time thinking through the user experience. It took a lot of testing and many different versions to get the user flow done well. I also found out that it was often important to talk to testers in person. Without talking to testers, it is difficult to understand why they are making certain choices. When I asked users to search along a route, I was able to ask where they would expect to find the feature. I was then able to try different options, and test them until the users found the flow easier. Discussing it with them helped to gather more information on how to improved the experience.
